#### 基本 Markdown 語法

所有標準 Markdown 語法都支援：

- **粗體** - 使用 `**text**` 或 `__text__`
- *斜體* - 使用 `*text*` 或 `_text_`
- ~~刪除線~~ - 使用 `~~text~~`
- `行內程式碼` - 使用反引號
- [連結](https://example.com) - `[text](url)`

### 建議的文章結構

一篇優秀的技術部落格文章通常包含：

1. **引言** (Introduction)
   - 吸引讀者注意
   - 說明文章主題
   - 列出讀者會學到什麼

2. **背景知識** (Background / Prerequisites)
   - 讀者需要的先備知識
   - 相關概念介紹

3. **主要內容** (Main Content)
   - 分成多個小節
   - 使用標題組織內容
   - 配合程式碼範例

4. **實作範例** (Practical Examples)
   - Step-by-step 教學
   - 完整可運行的程式碼

5. **注意事項** (Gotchas / Common Mistakes)
   - 常見錯誤
   - 最佳實踐

6. **總結** (Conclusion)
   - 回顧重點
   - 延伸閱讀
   - 下一步建議

### 撰寫風格建議

**語氣和風格：**
- 使用第一人稱（"I built this" 而不是 "This was built"）
- 保持對話式語氣，像是在跟朋友解釋
- 避免過度正式或學術化的用語
- 誠實分享經驗，包括失敗和學習

**內容品質：**
- 解釋「為什麼」，不只是「怎麼做」
- 提供實際可用的程式碼範例
- 包含截圖或示意圖幫助理解
- 連結到相關資源和文檔

**SEO 優化：**
- 標題包含關鍵字
- 使用描述性的 heading (h2, h3)
- 內文自然使用關鍵字
- 添加相關的內外部連結

**可讀性：**
- 段落保持簡短（2-4 句）
- 使用列表分解資訊
- 善用 Callout 區塊強調重點
- 程式碼加上註解說明
